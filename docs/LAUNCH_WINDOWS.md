# Запуск приложения на Windows

Эта страница описывает простой запуск Streamlit-приложения для тестировщика или пользователя без ручного ввода длинных команд.

Важно: `run_app_windows.bat`, `run_app_windows.ps1` и `ChargedTrapLauncher.exe` являются запускателями. Они запускают локальный проект из этой папки через `.venv`. Это не полноценный автономный exe со всем научным приложением внутри и не упаковка всего расчётного приложения в один большой файл.

## Вариант A. Самый простой запуск

1. Распакуйте ZIP с проектом.
2. Откройте папку проекта.
3. Дважды нажмите `run_app_windows.bat`.
4. Дождитесь создания `.venv` и установки зависимостей из `requirements.txt`.
5. Браузер должен открыться сам. Если он не открылся, перейдите вручную на:

```text
http://localhost:8501
```

Первый запуск может быть долгим, потому что зависимости устанавливаются локально в `.venv`.

Файл `run_app_windows.bat` специально оставлен ASCII-only: в нём нет русских сообщений, чтобы `cmd.exe` не ломал текст из-за кодировки. Русские пояснения находятся в этой инструкции.

## Вариант B. Запуск из PowerShell

Откройте PowerShell в папке проекта и выполните:

```powershell
.\run_app_windows.ps1
```

PowerShell-скрипт делает то же самое: создаёт `.venv`, устанавливает зависимости и запускает:

```powershell
.\.venv\Scripts\python.exe -m streamlit run app.py
```

Если политика выполнения PowerShell блокирует `.ps1`, используйте `run_app_windows.bat`.
Если `.bat` показывает странные ошибки кодировки или запуска, попробуйте PowerShell-вариант:

```powershell
powershell -ExecutionPolicy Bypass -File .\run_app_windows.ps1
```

## Вариант C. Сборка exe-запускателя по желанию

Exe-запускатель можно собрать локально:

```powershell
.\scripts\build_windows_launcher.ps1
```

После сборки запустите:

```powershell
.\dist\ChargedTrapLauncher.exe
```

Этот exe является только лёгким запускателем. Он не содержит внутри себя всё научное приложение, Python-зависимости, расчётные модули и данные. Обычно рядом должны оставаться файлы проекта, `requirements.txt`, `app.py` и `.venv`.

Если Windows SmartScreen ругается на самодельный exe, безопаснее запускать `run_app_windows.bat` или `run_app_windows.ps1`.

## Возможные проблемы

Если появляется ошибка про `streamlit`, `streamlit-drawable-canvas`, `PIL`, `numpy`, `scipy` или другой пакет, выполните:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Если порт `8501` занят, Streamlit может выбрать другой порт. В таком случае смотрите URL в консоли Streamlit и откройте его в браузере.

Если `.venv` повреждён или установка зависимостей прервалась, удалите папку `.venv` и снова запустите `run_app_windows.bat`.
