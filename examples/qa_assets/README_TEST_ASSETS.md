# QA-ассеты для ручного тестирования

Эта папка содержит готовые картинки и JSON-файлы для ручной проверки
Streamlit-интерфейса проекта `charged_particle_trap`.

## Как сгенерировать файлы

Из корня проекта выполните:

```powershell
.\.venv\Scripts\python.exe examples\qa_assets\generate_qa_assets.py
```

Или универсально:

```powershell
python examples/qa_assets/generate_qa_assets.py
```

Скрипт детерминированный: при повторном запуске создаёт те же самые файлы.

## Как использовать картинки в Streamlit

1. Запустите приложение:

```powershell
streamlit run app.py
```

2. Выберите источник поля `Рисунок электродов`.
3. Загрузите нужный PNG из этой папки.
4. Проверьте оригинальное изображение, бинарную маску, connected components.
5. Назначьте ролям компонентов `RF`, `DC`, `GND` или `CUSTOM`.
6. Запустите расчёт поля и траектории.
7. Сохраните скриншоты и экспортные файлы, если проверяете экспорт.

## Файлы

| Файл | Для какого теста | Ожидаемые компоненты | Что проверять |
|---|---|---:|---|
| `mask_four_rectangles.png` | Базовый image mask mode | примерно 4 | Загрузка маски, connected components, ручное назначение RF/DC/GND/CUSTOM, RF map, DC map, расчёт поля |
| `mask_ring.png` | Кольцевая геометрия | примерно 1 | Обработка области с отверстием, бинаризация, расчёт поля |
| `mask_asymmetric_rf_dc.png` | Ручные RF/DC роли | примерно 5 | Несимметричная геометрия, разные роли электродов, CUSTOM напряжение |
| `mask_many_small_components.png` | Проверка предупреждений и cleanup | больше 4 | Поведение при множестве мелких компонент, понятность warnings, устойчивость UI |
| `mask_black_electrodes_on_white.png` | Проверка polarity/threshold | зависит от threshold | Ситуация, где электроды тёмные, а фон светлый; нужно проверить, понятно ли пользователю, как настроить маску |
| `sample_canvas_design.json` | Canvas save/load | 4 | Загрузка готового canvas-дизайна, восстановление маски и ролей |
| `sample_experiment_config.json` | Experiment save/load | зависит от preset | Загрузка `experiment_config.json` во вкладке `Эксперименты` |

## Что считать нормальным

- Число компонент может немного отличаться, если пользователь меняет threshold,
  cleanup или downsampling.
- Для `mask_many_small_components.png` мелкие компоненты специально добавлены,
  чтобы проверить предупреждения и удобство интерфейса.
- Для `mask_black_electrodes_on_white.png` может понадобиться изменить
  threshold или polarity/интерпретацию маски. Если интерфейс не объясняет, что
  делать, это хорошее UX-замечание.

## Что прикладывать к баг-репорту

Если баг связан с ассетом, укажите:

- имя файла;
- выбранный режим;
- threshold и другие настройки маски;
- роли электродов;
- скриншот preview;
- текст ошибки из Streamlit или PowerShell.

Пример:

```text
Файл: examples/qa_assets/mask_many_small_components.png
Режим: Рисунок электродов
Проблема: после загрузки найдено много компонент, но предупреждение непонятно.
```
